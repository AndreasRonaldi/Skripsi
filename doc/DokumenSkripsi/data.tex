\abstrakINA{Pada Universitas Katolik Parahyangan, digunakan SharIF Judge sebagai Online Judge yang telah dimodifikasi untuk mendukung kebutuhan pembelajaran. Namun, sistem ini masih memiliki keterbatasan dalam pengawasan, terutama saat ujian online, di mana potensi kecurangan seperti \textit{copy-paste} meningkat. Untuk mempermudah pengawasan, tugas akhir sebelumnya telah mengintegrasikan Integrated Development Environment atau IDE ke dalam SharIF Judge, memungkinkan seluruh proses pengerjaan tugas dilakukan dalam satu platform.

Meskipun demikian, IDE tersebut tidak dapat mengawasi aktivitas pengguna jika terjadinya kecurangan. Oleh karena itu, dalam tugas akhir ini, dikembangkan fitur perekaman ketikan dan pemutaran ulang ketikan pada IDE SharIF Judge. Fitur ini bertujuan memudahkan pengawasan, mendeteksi kecurangan, dan memberikan bukti jika terjadi pelanggaran.

Fitur ini akan dirancang untuk merekam seluruh aktivitas mahasiswa yaitu perubahan kode, navigasi keluar \textit{web page}, dan aksi pada IDE seperti menyimpan kode program. Setelah itu, data akan disimpan dalam format JSON agar dapat divisualisasikan kembali kepada pengajar. 

Implementasi fitur ini diuji secara fungsional dan eksperimental. Hasil pengujian menunjukkan bahwa sistem mampu merekam aktivitas mahasiswa secara rinci dan memutar ulang proses pengerjaan secara akurat. Analisis terhadap data rekaman juga mengungkapkan pola-pola pengerjaan yang wajar maupun mencurigakan, seperti tingkat perubahan kode, keberadaan debugging, waktu jeda berpikir, serta indikasi copy-paste.

Sistem ini memberikan dukungan pengawasan tambahan yang efektif, meskipun belum menggunakan metode pendeteksian otomatis. Kesimpulan dari penelitian ini adalah bahwa fitur pemutaran ulang ketikan dapat membantu dosen dalam melakukan evaluasi yang lebih objektif. Saran pengembangan selanjutnya mencakup otomatisasi deteksi pola, efisiensi penyimpanan, dan peningkatan antarmuka pengguna.}
\abstrakENG{At Parahyangan Catholic University, SharIF Judge is used as an Online Judge that has been modified to support learning needs. However, this system has limitations in monitoring especially during online exams, where the potential for cheating such as copy-pasting will increases. To fix the problem, the previous final project integrated an Integrated Development Environment or IDE into SharIF Judge, allowing the entire task completion process \mbox{to be conducted within a single platform}.

Nevertheless, the IDE could not monitor user activity in cases of cheating. Therefore, in this final project, a keystroke recording and playback feature was developed for the SharIF Judge IDE. This feature aims to simplify monitoring, detect cheating, and provide evidence of violations.

The feature is designed to record all student activities, including code changes, navigation away from the web page, and actions within the IDE such as saving program code. The data is then stored in JSON format so it can be visualized for instructors.

The implementation of this feature was tested functionally and experimentally. The test results showed that the system was able to record student activities in detail and replay the work process accurately. Analysis of the recorded data also revealed reasonable and suspicious work patterns, such as the level of code change, the presence of debugging, thinking pauses, and indications of copy-paste.

This system provides effective additional supervision support, although it does not yet use an automatic detection method. The conclusion of this study is that the typing replay feature can help lecturers in conducting more objective evaluations. Suggestions for further development include pattern detection accuracy, storage efficiency, and user interface improvements.}
